# Code Review 检查清单

## 📋 通用检查清单

### 功能正确性

- [ ] **需求实现**: 代码实现了所有需求功能
- [ ] **边界情况**: 处理了空值、null、0、-1、空数组等边界
- [ ] **错误处理**: 完整的 try-catch，错误日志
- [ ] **返回值**: 返回值类型正确，处理了所有分支
- [ ] **副作用**: 清楚所有副作用（数据库修改、文件写入、网络请求）

### 代码质量

- [ ] **函数长度**: 函数不超过 30 行
- [ ] **命名清晰**: 变量/函数名清晰表达意图
- [ ] **单一职责**: 每个函数/类只有一个职责
- [ ] **无重复**: 没有重复代码（DRY 原则）
- [ ] **注释恰当**: 注释解释 why 不是 what

### 安全性

- [ ] **SQL 注入**: 使用参数化查询，不拼接 SQL
- [ ] **XSS**: 用户输入转义，不使用 innerHTML
- [ ] **认证**: 密码哈希存储（bcrypt），不存明文
- [ ] **授权**: 每层都验证权限，不信任客户端
- [ ] **敏感数据**: 不日志密码/Token，HTTPS 传输
- [ ] **文件上传**: 验证 MIME 类型，限制大小，重命名

### 性能

- [ ] **N+1 查询**: 循环内无数据库查询
- [ ] **缓存**: 热点数据有缓存
- [ ] **内存**: 无内存泄漏，缓存有上限
- [ ] **I/O**: 使用异步非阻塞 I/O
- [ ] **计算**: 无重复计算，昂贵操作有缓存

### 测试

- [ ] **单元测试**: 有关键逻辑的单元测试
- [ ] **边界测试**: 测试了边界情况
- [ ] **错误测试**: 测试了错误路径
- [ ] **测试通过**: 所有测试通过

### 文档

- [ ] **README**: 更新了 README（如有新功能）
- [ ] **API 文档**: 更新了 API 文档
- [ ] **注释**: 复杂逻辑有注释说明

## 🔍 语言特定检查清单

### JavaScript/TypeScript

- [ ] **类型安全**: TypeScript 类型定义正确
- [ ] **async/await**: async 函数正确使用 await
- [ ] **Promise 处理**: Promise 有 catch 处理
- [ ] **数组方法**: forEach 内不用 async（用 for...of 或 Promise.all）
- [ ] **空值处理**: 使用可选链（?.）和空值合并（??）
- [ ] **变量声明**: 使用 const/let，不用 var
- [ ] **相等判断**: 使用 === 而不是 ==
- [ ] **事件监听**: 移除不再需要的事件监听器

### Python

- [ ] **类型提示**: 有类型提示（Type Hints）
- [ ] **异常处理**: try-except 捕获特定异常
- [ ] **资源管理**: 使用 with 语句管理资源
- [ ] **列表推导**: 适当使用列表推导式
- [ ] **导入**: 导入在文件顶部，无循环导入
- [ ] **PEP 8**: 遵循 PEP 8 风格指南
- [ ] **GIL 感知**: CPU 密集型用多进程，IO 密集型用多线程/asyncio

### Java

- [ ] **空指针**: 检查 null，使用 Optional
- [ ] **资源关闭**: 使用 try-with-resources
- [ ] **集合类型**: 使用泛型，不使用原始类型
- [ ] **异常**: 不吞异常，记录日志或抛出
- [ ] **并发**: 正确使用 synchronized 或并发容器
- [ ] **日志**: 使用日志框架，不用 System.out
- [ ] **常量**: 魔术数字提取为常量

### Go

- [ ] **错误处理**: 检查每个 error 返回值
- [ ] **defer**: 使用 defer 清理资源
- [ ] **goroutine**: 不泄露 goroutine，有退出机制
- [ ] **channel**: channel 操作有 select 超时
- [ ] **接口**: 接口设计小而精
- [ ] **指针**: 避免不必要的指针
- [ ] **init**: 避免复杂的 init 函数

## 🐛 Bug 模式检查

### 空指针/Undefined

- [ ] 对象属性访问前检查 null/undefined
- [ ] 数组访问前检查长度
- [ ] 函数返回值可能为 null 的情况已处理

### 类型错误

- [ ] 数字运算前确认是 number 类型
- [ ] 字符串拼接前确认是 string 类型
- [ ] 函数参数类型与声明一致

### 异步错误

- [ ] async 函数内调用 Promise 已 await
- [ ] forEach 内没有用 await（改用 for...of）
- [ ] 错误处理有 catch

### 资源泄漏

- [ ] 文件操作后关闭
- [ ] 数据库连接释放
- [ ] 网络连接关闭
- [ ] 使用 try-finally 或 with

### 竞态条件

- [ ] 共享状态并发访问已保护
- [ ] 数据库事务隔离级别正确
- [ ] 前端防止重复提交


[有问题的代码]


## 🎯 使用指南

### Code Review 流程

1. **准备阶段**
   - 阅读 PR 描述
   - 了解改动范围
   - 运行测试

2. **第一轮审查**
   - 使用检查清单
   - 标记问题
   - 写评论

3. **反馈阶段**
   - 与作者沟通
   - 澄清疑问
   - 确认修复方案

4. **验证阶段**
   - 检查修复
   - 运行测试
   - 批准或要求重审

5. **合并后**
   - 持续监控
   - 收集反馈
   - 改进检查清单
